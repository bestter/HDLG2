Refactor IFilePropertyGetter to accept FileInfo instead of string path

Passed the existing `FileInfo` object yielded from directory enumeration downstream through the `FilePropertyBrowser` and into individual `IFilePropertyGetter` implementations. This prevents each getter from redundantly instantiating `new FileInfo(path)` just to check properties like `Length`, which forces the OS to re-stat the file. Also resolved the remaining CA1062 issue by checking for null FileInfo before logging.
