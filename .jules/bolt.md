## 2026-07-23 - Avoid Double I/O Enumeration and Respect Iteration Filters
**Learning:** When attempting to optimize file system enumeration in WinForms tree loading, replacing a single `GetFileSystemInfos` pass with separate `EnumerateDirectories` and `EnumerateFiles` is an anti-pattern. It forces a redundant disk traversal (since Windows underlying APIs return both files and directories together). Furthermore, separating loops requires duplicating early-exit filter logic (like skipping symlinks).
**Action:** When gathering UI nodes, use a single fast I/O pass (like `GetFileSystemInfos`) to fetch raw data. Optimize on the CPU-side by pre-allocating an exactly-sized single List or Array (`fsInfos.Length`) to prevent `List<T>` dynamic resizing overhead, and iterate over the local array twice (first for directories, second for files) to achieve proper visual sorting before executing a single UI `AddRange` call.
