🎯 **What:** The `OpenWithDefaultProgram` method automatically opens files using the system's default handler without giving the user a chance to review the action, relying purely on a blocklist of dangerous extensions.

⚠️ **Risk:** By delegating execution to `ShellExecute` without user confirmation, the application may inadvertently open files that bypass the blocklist or exploit zero-day shell execution vulnerabilities. If a user unknowingly clicks a link or opens a file from an untrusted source, the application would automatically execute it.

🛡️ **Solution:** Added a `MessageBox` prompt before the `Process.Start` call. This gives the user a clear security warning about opening files from untrusted sources and requires their explicit consent ("Yes") before proceeding to open the file with the default program.
